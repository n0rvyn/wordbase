## [S1] Specific Error Candidates

**[断言 1]** Task 3 step 3: `define:vars={{ app: appC, app2 }}` produces CSS variable names `--app` and `--app2`, but the reference CSS and `tokens.css:77` use the hyphenated name `--app-2`. The ported styles in Task 2 will reference `var(--app-2)` — which resolves to nothing, so the darkened app color never appears.

验证：
- `tokens.css:77–78`: "App surfaces additionally set `--app` / `--app-2` from the app's own brand color" — hyphen confirmed.
- Reference `App Detail.html:377`: `document.documentElement.style.setProperty('--app-2', shade(d.accentColor,-28))` — hyphenated.
- `define:vars` in Astro emits `--{key}` verbatim. Key `app2` (no hyphen) → `--app2`. Key `--app-2` needs to be spelled with the hyphen in the JS object, which is not valid as a bare identifier → must be quoted: `define:vars={{ 'app': appC, 'app-2': app2 }}`.
- Additionally, Task 3's inner remap `{ --app: var(--app); --app-2: var(--app-2) }` is circular: `--app: var(--app)` is a self-reference (infinite loop / undefined in CSS), and `--app-2: var(--app-2)` reads the non-existent `--app2` (no hyphen). The "`:root scope or wrapper`" wording leaves the element target ambiguous.

結果：❌ 断言成立 [C:97] → 双色 `--app-2` 永远为空，Task 2 ported CSS 的所有 `var(--app-2)` 规则失效。

修订：Task 3 step 3 必须修正为：
```astro
<main style={`--app:${appC};--app-2:${app2}`}>
```
or, if sticking to `define:vars`, use quoted keys:
```astro
<style define:vars={{ app: appC, 'app-2': app2 }}>
```
...and remove the circular self-referencing remap inside the `<style>` block entirely (define:vars already inlines the values as `--app: value; --app-2: value` on the element).
