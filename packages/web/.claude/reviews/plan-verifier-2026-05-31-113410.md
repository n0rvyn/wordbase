### Item 1 — CSS var naming and inheritance (define:vars → inline style)

**Fix applied:** Task 3 step 3 uses `<main id="top" style={`--app:${appC};--app-2:${app2}`}>` instead of `define:vars`.

**(a) Var names match ported CSS:**
Confirmed. The inline style emits `--app` and `--app-2` (with hyphen), matching `var(--app)` and `var(--app-2)` used throughout `app-detail.css` (ported from reference lines 103, 114, 122, 144, 150, 151). Astro `define:vars={{app2}}` would have emitted `--app2` (no hyphen) — silently broken. Inline style emits verbatim key, so `--app-2` resolves correctly.

**(b) CSS inheritance to descendants:**
Confirmed. CSS custom properties inherit through the DOM tree. Any element with `style="--app:...; --app-2:..."` passes those values to all descendant elements via CSS inheritance. `.ad-hero`, `.ad-icon`, `.shot-frame`, `.screen`, `.sf-ring`, `.sf-pill` are all descendants of the element carrying the inline style, so `var(--app)` / `var(--app-2)` resolve correctly there. No scoping issue.

**(c) No leak to other pages / BaseLayout override:**
BaseLayout (`src/layouts/BaseLayout.astro`) does not define `--app` or `--app-2` in CSS. It only sets `--accent` via JS (`document.documentElement.style.setProperty('--accent', accent)` — lines 56, 128). The reference's static CSS defines `--app:#0E8C7A; --app-2:#0A5C52` on `:root` (reference line 30), but that is not ported to `tokens.css` (tokens.css line 77 is a comment only). So other Astro pages have no inherited `--app`/`--app-2` at all — no conflict.

**NEW INCONSISTENCY FOUND — must-revise:**
BaseLayout already renders `<main><slot /></main>` (BaseLayout.astro line 81-83). The plan's `<main id="top" style={...}>` would be slotted inside that `<main>`, creating nested `<main>` elements — invalid HTML5 (spec: there must be no more than one `<main>` element visible). Existing pages use `<section id="...">` or bare `<div>` as their top-level slot content (confirmed: index.astro uses `<section class="sec wrap" id="apps">` etc.; writing/index.astro uses `<div class="row2">`).

The fix is trivial: change `<main id="top" style={...}>` to `<div id="top" style={...}>`. CSS var inheritance is unaffected — `div` is a valid CSS custom property scope. Semantics preserved: `id="top"` anchor still works; `--app`/`--app-2` still inherit to all descendants.

**Verdict: NOT CONFIRMED as written. The CSS-var mechanism is correct, but the element type must be `<div>`, not `<main>`.**
