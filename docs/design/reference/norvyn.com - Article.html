当一座博客学会三种语气

重做 norvyn.com 的全过程：如何把博客、作品集与播客，收进同一套会呼吸的设计语言里，让它们既各有性格，又像出自同一只手。

N
norvyn · 2026 年 5 月 21 日

三年前我写下 norvyn.com 的第一行 HTML 时，它只是一个放文章的地方。后来它慢慢长出了别的东西：我做的 App 要有地方展示，我录的播客要有地方收听。于是同一个域名下，住进了三种完全不同的内容——而它们看起来，像是三个互不相识的网站拼在一起。

这次重做，我想回答一个问题：一个人的作品集，能不能既有三种声音，又像是出自同一只手？

三种声音，一只手

博客是安静的，适合长时间停留；App 落地页是外向的，要在几秒内说清楚价值；播客是亲密的，像有人在你耳边说话。它们的语气本就不同，强行统一只会让每一个都变得平庸。

我的答案是：统一语法，而不是统一语气。三种内容共享同一套排版骨架、同一种留白节奏、同一组颜色令牌；但在这套语法之上，各自说各自的话。

统一的不是长相，是教养。三个孩子可以性格迥异，但你一眼能看出他们是一家人。

先有系统，再有页面

我没有从首页开始画。我先定义了一套语义令牌——不是「这个蓝色」，而是「强调色」；不是「16 像素」，而是「正文字号」。页面只是这些令牌的一次具体排列。

这样做的好处是，换肤变成了一件几乎免费的事。深色模式不是重画一遍，而是同一套令牌换一组值：

/* 同一个变量，两套值 */
:root[data-theme="light"]{ --ink: #15151A; }
:root[data-theme="dark"] { --ink: #F3F2F0; }

页面里的每一处文字都写 color: var(--ink)。我再也不用关心此刻是白天还是黑夜——令牌替我记得。

为什么是线，而不是盒子

早期的版本里，每个区块都是一张带阴影、带圆角的卡片。堆在一起，页面像贴满了膏药。后来我把几乎所有的卡片都拆了，只留下一根 1 像素的线。

  • 线不抢戏，它只负责把内容分开；
  • 线让密度可以很高，而不显得拥挤；
  • 线天然地把目光，引导向真正有颜色的东西——也就是作品本身。

「克制」不是少做，而是把每一次「多做」的冲动，都先放一晚上。

把颜色让给作品

整个站点的「外壳」几乎是单色的：近白、近黑、几级灰，加上一个克制的强调色。这不是因为我不喜欢颜色，恰恰相反——我想让唯一的颜色，来自我真正在乎的东西：App 的图标、截图，播客的封面。

当背景退到几乎无色，一个绿色的 App 图标会自己亮起来。这是一种把舞台让给主角的设计。

然后，把它做出来

系统再漂亮，不上线就等于零。这一版我给自己定了一条规矩：能砍的都砍掉。上线前，我删掉了一半已经写好的功能——评论、标签云、阅读统计、花哨的页面转场。

留下的，是一个加载很快、读起来很安静、三种内容各得其所的地方。它没有那么多功能，但每一个还在的功能，我都确信它该在。

这大概就是我想要的 norvyn.com：一座学会了三种语气，却始终用同一只手写字的房子。
